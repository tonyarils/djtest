# 产品需求文档 (PRD)：“智慧红芯”党建协同管理系统

**文档版本**：V1.0
**产品定位**：基于工单生命周期管理的自动化、智能化基层党建协同平台。

## 1. 业务概述与目标
针对基层党建工作中普遍存在的“任务落实难溯源、数据分析靠人工、会议记录不规范”等痛点，本系统旨在通过构建“任务工单化、执行写实化、考核数据化”的业务闭环，提升组织管理效能，减轻基层负担。

## 2. 总体组织与角色架构
系统严格映射真实党内组织关系，采用**“党委 -> 支部 -> 党小组 -> 党员”**的四级树状结构。取消泛化的“负责人”概念，实行基于标准党内职务的视图与权限隔离：

* **党委/支部书记**：拥有本级及下属层级工单的终审权、异常工单重派权及全局数据看板查看权。
* **组织委员**：负责日常任务工单的派发、党小组/党员的执行者分配以及进度督办。
* **纪检委员**：拥有系统级审计日志（Audit Log）查阅权限，负责处理超时红名预警与任务质量抽检。
* **宣传委员**：主导会议纪要审核、多模态附件（音视频/AI转写）提取，及对外宣传物料的带水印导出。
* **群工委员**：统筹志愿服务类任务的发布与协调。
* **动态执行者**：党员或党小组在被指派任务后，自动激活当前工单的“执行者”状态，获取凭证上传入口。

## 3. 核心功能模块设计

### 3.1 任务工单引擎 (Task Engine)
任务下发与流转的自动化核心，所有工作均以“工单”形式存在，状态包含：`待领用`、`处理中`、`审核中`、`已结项`、`已挂起`。
* **A类任务（即时指令）**：由上级手动创建，支持一键下发至指定层级或特定党员，并指定响应时间。
* **B类任务（周期常规）**：如“三会一课”，系统按设定周期（如周/月/季）循环生成待办工单。
* **C类任务（计划触发）**：由系统根据年度党建计划表，于每月 1 日凌晨 00:00 自动生成实例化工单，并精准推送到对应支部。
* **自主抢单模式**：针对志愿服务或开放性活动，支持党员在移动端自主认领工单。

### 3.2 预警与评价闭环 (Warning & Evaluation)
建立“事前提醒、事后评估”的闭环机制。
* **红名预警机制**：工单在截止日期前 3 天若仍处于 `PENDING` 状态，系统自动将其 `warning_level` 设为 1，列表红色高亮，并向同级纪检委员与书记推送（支持企业微信/WebSocket秒级提醒）。
* **星级评价流**：任务进入“审核中”后，审核委员需对“执行时效性、写实完整度、成果质量”进行综合评定（1-5星）。
* **黄名末位督办**：月度平均得分低于 3 星的支部自动触发“黄名”标签，高分工单则自动换算为年度先锋积分。
* **申诉通道**：允许下级对低星评价发起申诉工单，交由上一级党委复核。

### 3.3 会议与写实管理 (Meeting & Real-record)
* **会议一键流转**：可由B/C类工单直接发起会议，自动分发会议通知。
* **防伪签到**：移动端 LBS 定位结合人脸识别（活体检测），杜绝代签。
* **多模态与 AI 转写**：
    * 支持现场上传高清图片、原声录音及视频。
    * 接入 ASR（自动语音识别）大模型，将现场语音附件自动转写为结构化文字纪要，并自动提取核心摘要及关键词。
* **决议转任务**：会议形成的待办事项，支持一键拆解为子工单并定向派发。

### 3.4 数据资产与安全归档 (Assets & Security)
* **一任务一档案**：依据工单编号、执行年份、所属支部自动建立电子卷宗，集中管理多模态附件与业务表单。
* **动态水印与脱敏**：任何角色在导出人员花名册、积分表或敏感附件时，系统强制追加“组织名称+职务+姓名+时间”的隐形及显性双层动态水印。输出报表需支持导出为规范的 Word (.docx) 文档格式。

### 3.5 党员信息管理
系统需提供完整的党员信息维护能力，用户表至少包含以下信息字段：
* **姓名**
* **工作证号**
* **出生年月**
* **民族**
* **性别**
* **学历**
* **入党时间**
* **参加工作时间**
* **组织**
* **党内职务**
* **岗位职务**
* **转入时间**
* **转出时间**
* **转出原因**
* **备注**

## 4. 数据分析指标与看板

为党委决策驾驶舱提供以下核心计算指标：
1. **任务销号率** = 完成并结项的工单总数 / 下发工单总数 （衡量完整性）
2. **响应时长** = 工单状态变为处理中的时间戳 - 工单下发的时间戳 （衡量敏捷度）
3. **活动写实度** = 包含音视频或图片附件的工单数 / 需现场执行的工单总数 （衡量防伪性）
4. **满意度均值** = 工单评价总星数 / 已评价工单总数 （衡量执行质量）

## 5. 非功能性与技术要求
* **性能优化**：针对管理驾驶舱的多维度数据下钻，需对 SQL 时间密度分析等高耗时查询进行索引级优化。
* **权限安全**：在数据库底层设计时，需严格管控 `CREATE ANY TABLE` 等高危权限的分配，确保各级委员的视图操作在最小特权原则下运行。
* **存储架构**：针对海量会议视频和图片附件，需采用对象存储（OSS/MinIO）进行读写分离，关系型数据库仅存储文件索引及业务流转数据。
